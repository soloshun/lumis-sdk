# OpenARIA governance

OpenARIA currently uses a benevolent-maintainer model. Solomon Eshun is the project maintainer and release authority.

## Decisions

Small, reversible changes use issues and pull requests. Cross-cutting changes require an RFC before implementation, including new core ports, breaking domain or configuration changes, plugin-system changes, execution capability, default model behavior, remote telemetry, and governance or licensing changes.

Accepted architectural decisions should be recorded in `docs/adr/` as the contributor base grows.

## Contribution and ownership

Contributions remain subject to the Apache-2.0 license and clean-room policy.

## Releases

Only maintainers publish releases. Release credentials must not be shared through issues, pull requests, or repository files. Pre-1.0 contracts may evolve with documented compatibility and migration notes.
