# Lumis SDK 1.0 release gate

Status: **not approved for stable release**

This decision record separates work the repository can verify from evidence that requires an
independent person or review. It is the checklist for a future `1.0.0rc1` and `1.0.0` release; it
does not itself publish either artifact.

## Release ownership

The release maintainer assembles evidence, rehearses upgrades and plugins, and prepares the
release pull request. The lead maintainer named in `GOVERNANCE.md` owns the final stable go/no-go
decision. Any waiver must name the residual risk, bounded scope, owner, expiry/review date, and a
public follow-up issue.

## Local gate

- [x] Stable public API/configuration inventory and v1 migration tooling.
- [x] Compatibility and deprecation policy with executable contract tests.
- [x] Internal threat-model, authority, hostile-input, and residual-risk review.
- [x] SBOM, provenance workflow, reproducible-build comparison, artifact inspection, and clean
      Python 3.11–3.13 installation checks.
- [x] Core/official-plugin coexistence checks and public plugin contract suite.
- [x] Navigable documentation, standalone paths, offline cookbook smoke tests, and written video
      equivalent.
- [x] Public governance, support boundaries, contributor provenance, plugin ownership, and
      release ownership.
- [x] Reproducible bounded benchmark methodology.
- [ ] Set an RC version, build `1.0.0rc1`, and attach the exact commit, lockfile digest, artifact
      digests, SBOM, attestations, CI run, and benchmark JSON.
- [ ] Rehearse upgrade from every public pre-1.0 configuration shape and latest PyPI release in a
      clean environment; record commands and results below.
- [ ] Install every official plugin combination against the RC and run package contract suites;
      record versions, commands, and results below.

## Required external gate

- [ ] Independent security review received; findings are resolved or explicitly accepted with
      bounded residual risk.
- [ ] At least two non-Lumis adopters reproduce the documented core-only and plugin paths and
      provide feedback attributable to a person/project with permission to publish it.
- [ ] RC feedback window closes with no unresolved critical/high issue or authority ambiguity.

Maintainer-authored examples prove reproducibility, not independent adoption. CI security tools
support review, but do not substitute for an independent security assessment.

RC rehearsal record, independent security review, or independently attributable adopter
