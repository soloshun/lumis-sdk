This record captures reproducible evidence for the Phase 1 candidate. The complete dated go/no-go

assert the authoritative package `__version__`, making future release bumps testable without
weakening version verification. The complete gate was rerun and passed after that fix.

## Public RC qualification

- The exact core commit `fa516e87dce2f4e2ae80abca31f341f2d8a2309d` was published as
  [`v0.1.0rc1`](https://github.com/soloshun/lumis-sdk/releases/tag/v0.1.0rc1), qualified on
  [TestPyPI](https://github.com/soloshun/lumis-sdk/actions/runs/29813044609), and published through
  the trusted [PyPI workflow](https://github.com/soloshun/lumis-sdk/actions/runs/29813168232).
- Clean public-index verification passed import, CLI version, initialization, doctor, and rule
  validation. Wheel and source distributions also clean-installed on Python 3.11, 3.12, and 3.13.
- The public `0.0.8` to `0.1.0rc1` rehearsal migrated the synthetic project/rule fixture from
  `lumis.dev/v1alpha1` to `lumis.dev/v1`; doctor passed and JSON rule validation returned
  `valid: true`.
- The first public plugin rehearsal found that immutable `0.1.1` wheels retained a `<0.1.0` core
  dependency bound. PRs [#67](https://github.com/soloshun/lumis-sdk/pull/67) and
  [#68](https://github.com/soloshun/lumis-sdk/pull/68) republished both plugins as `0.1.2`.
  A clean PyPI-only install then reported exactly the PostgreSQL and HTTP plugins as compatible,
  without importing either implementation or granting authority.
- The retained [SPDX SBOM](https://github.com/soloshun/lumis-sdk/actions/runs/29813168232/artifacts/8488252997),
  [build provenance](https://github.com/soloshun/lumis-sdk/attestations/36307635), and
  [SBOM attestation](https://github.com/soloshun/lumis-sdk/attestations/36307638) bind the public
  core artifacts to the release workflow.
